## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (gap-1) |
| sm | 8px | Compact element spacing (gap-2, p-2) |
| md | 16px | Default element spacing (p-4, gap-4, mb-4) |
| lg | 24px | Section padding (p-6, mb-6) |
| xl | 32px | Layout gaps (p-8, gap-8) |
| 2xl | 48px | Major section breaks (py-12) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: Touch targets for quiz option buttons must be minimum 48px tall (p-4 with text achieves this). SRS rating buttons (Again/Hard/Good/Easy) must be minimum 48px wide for tap accuracy.

## Interaction Contracts

### Quiz Mode Selection Flow
1. User taps "Quiz" tab in top navigation
2. QuizModeSelector renders with two mode cards
3. Each card shows mode name, description, and due count (via DueCountBadge)
4. User taps a mode card -- border transitions to accent-600, 150ms ease
5. View transitions to quiz session with selected mode

### Vocabulary Population Flow
1. User taps "Add Words to Quiz" in KaavyaReader toolbar or KaavyaLibrary card
2. Button shows spinner (animate-spin) and disables
3. API analyzes text in chunks; no progress percentage needed (just spinner)
4. On success: button briefly shows green checkmark (check-appear animation, 400ms), then returns to default state with updated word count
5. On error: button returns to default, toast-style inline error below button (text-sm text-red-600, fade-in animation)

### SRS Rating Flow (replaces auto-continue)
1. User taps an answer option (existing SELECT behavior)
2. Feedback appears: correct/wrong message, correct answer if wrong, all meanings list
3. Below feedback: SRSRatingBar with 4 buttons showing next review intervals
4. User taps a rating -- card state updates in IndexedDB, quiz advances to next question
5. If user does not rate within 10 seconds, default to "Good" rating and auto-advance (with 1s warning flash on Good button)

### Mastery Dashboard Flow
1. VocabularyDashboard appears above QuizModeSelector
2. Stats update reactively via useLiveQuery
3. Mastery timeline text only appears after 5+ total reviews across all cards
4. Tapping a stat (e.g., "Due Now: 12") navigates to quiz with that filter pre-selected

### Empty States
- No vocabulary populated yet: VocabularyDashboard shows "0" for all stats with empty state message below
- No cards due: Mode card shows "All caught up" instead of due count, mode card border becomes dashed parchment-200
- Quiz session with 0 due cards: Redirect back to mode selector with "No words due for review" inline message
