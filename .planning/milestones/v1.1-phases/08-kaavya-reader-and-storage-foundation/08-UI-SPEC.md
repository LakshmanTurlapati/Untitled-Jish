## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, reader margins |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical breaks, touch targets (reader nav arrows) |
| 3xl | 64px | Reader viewport top/bottom clearance |

Exceptions: None. Reader page navigation touch targets use 48px (exceeds WCAG 44px minimum).

---

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px | 400 (regular) | 1.5 | --font-sans |
| Reader body | 20px | 400 (regular) | 2.0 | --font-sanskrit (Shobhika) |
| Label | 14px | 600 (semibold) | 1.4 | --font-sans |
| Heading | 24px | 600 (semibold) | 1.2 | --font-sans |

Notes:
- Reader body uses Shobhika at 20px with line-height 2.0 for comfortable extended Sanskrit reading (matches research recommendation and existing --spacing-sanskrit: 1.8 token, rounded up for readability).
- All UI chrome (buttons, labels, metadata) uses system sans-serif.
- Only 2 weights used: 400 (regular) and 600 (semibold). A 24px heading at semibold reads as a clear heading against 16px regular body text without requiring a third weight.

## Layout Contract

### Library View (READ-06)

**Focal Point:** The "Add Kaavya" CTA button in the top bar -- draws the user toward adding their first kaavya (empty state) or growing their collection (populated state).

```
+------------------------------------------+
|  [Library]  [+ Add Kaavya]               |  <- Top bar with heading + CTA
+------------------------------------------+
|                                          |
|  +----------------+  +----------------+  |
|  | Kaavya Title   |  | Kaavya Title   |  |  <- 2-column grid on desktop
|  | Author (opt)   |  | Author (opt)   |     1-column on mobile
|  | Page 3 of 24   |  | Page 1 of 12   |  |
|  | Last read: 2d  |  | Added: today   |  |
|  +----------------+  +----------------+  |
|                                          |
|  +----------------+  +----------------+  |
|  | ...            |  | ...            |  |
|  +----------------+  +----------------+  |
|                                          |
+------------------------------------------+
```

- Grid: `grid grid-cols-1 sm:grid-cols-2 gap-4`
- Card: parchment-100 background, parchment-200 border, 16px padding, 8px border-radius
- Card hover: slight shadow lift (shadow-md transition)

### Uploader View (READ-01, READ-02)

+------------------------------------------+

+------------------------------------------+
|  Title: [________________________]       |
|                                          |
|  +----------------------------------+   |
|  |                                  |   |
|  |    [PDF icon]                    |   |  <- Drag-and-drop zone
|  |    Drop PDF here or click to     |   |     OR textarea toggle
|  |    browse                        |   |
|  |                                  |   |
|  +----------------------------------+   |
|                                          |
|  -- or --                                |
|                                          |

|  |  Paste Sanskrit text here...     |   |  <- Textarea for paste
|  |                                  |   |

- Toggle between PDF upload and paste via tab-style switcher

### Reader View (READ-03, READ-04, READ-05)

**Focal Point:** The Sanskrit text block in the center of the reader viewport -- the primary reading surface occupies the visual center with all chrome pushed to edges.

```
+------------------------------------------+
|  [< Library]    Kaavya Title     [pg/pg] |  <- Minimal top bar
+------------------------------------------+
|                                          |
|                                          |
|    Sanskrit text rendered in Shobhika    |
|    at 20px with line-height 2.0.         |
|                                          |
|    Users can select shloka text to       |
|    trigger the interpretation panel.     |
|                                          |
|                                          |
+------------------------------------------+
|  [<]     * * o * *     [>]               |  <- Page nav: arrows + dots
+------------------------------------------+

When shloka selected, bottom sheet slides up:

+------------------------------------------+
|  Your Interpretation                     |
|  +----------------------------------+   |
|  |  Type your understanding...      |   |
|  +----------------------------------+   |
|  [Submit Interpretation]                 |
+------------------------------------------+
|  Hints from Pramaana                     |
|  > Consider the dhatu "gam" (to go)...  |
|  > Mallinatha's commentary suggests...  |
+------------------------------------------+
```

- Reader viewport: height 70vh, overflow hidden, parchment-50 background
- Page navigation: arrow buttons (48px touch target), dot indicators
- Active page dot: accent-500, inactive dots: parchment-200
- Page turn animation: translateX with 300ms ease transition
- Interpretation panel: slides up from bottom, parchment-100 background
- Hint items: left-bordered with accent-500 (4px left border), 16px padding

## Interaction Contracts

### Page Navigation (Reader)
- Left/right arrow buttons for page turn
- Keyboard: ArrowLeft/ArrowRight for page navigation
- Swipe gesture: optional (CSS scroll-snap or touch event handling)
- Page indicator dots show current position (max 7 visible, collapse with ellipsis for longer texts)

### Shloka Selection (Reader)
- User selects text in the reader viewport
- On selection, a floating "Interpret This" button appears near selection
- Tapping "Interpret This" opens the interpretation panel at bottom
- Selected shloka text is highlighted with accent-500 at 15% opacity background

### AI Hint Streaming (HintPanel)
- Hints stream in with a typing indicator (3 animated dots)
- Each hint appears with a fade-in animation (300ms)
- Hints are prefixed with ">" character in a blockquote style
- Loading state: pulsing skeleton lines

### Drag-and-Drop Upload
- Dropzone border changes to accent-500 (solid) when file is dragged over
- Background changes to accent-500 at 10% opacity during drag
- Upload progress shown as inline text: "Extracting text from PDF..."
- After extraction, preview of extracted text shown in read-only Shobhika textarea

### Delete Kaavya
- Swipe-to-delete on mobile (optional), explicit delete icon on hover for desktop
- Confirmation dialog: "This will permanently remove [title] from your library. Reading progress will be lost."
- Delete button in dialog: red-600 background

## Accessibility Notes

- Reader page navigation arrows: `aria-label="Previous page"` / `aria-label="Next page"`
- Page dots: `role="tablist"` with each dot as `role="tab"` and `aria-selected`
- Dropzone: `role="button"` with `aria-label="Upload PDF file"`
- Shloka selection floating button: focus-managed, appears on text selection
- Delete confirmation modal: focus-trapped, Escape to dismiss
- All interactive elements: minimum 48px touch target (exceeds WCAG 44px minimum)
- Reader text: minimum 20px for comfortable reading
- Color contrast: ink-800 (#2a1f14) on parchment-50 (#fdf8f0) exceeds WCAG AAA (contrast ratio >7:1)
