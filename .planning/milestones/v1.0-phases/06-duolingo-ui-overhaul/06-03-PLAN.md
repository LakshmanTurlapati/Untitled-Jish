**State management — useReducer pattern:**
```typescript
interface QuizGameState {
  questions: QuizQuestion[];
  currentIndex: number;
  phase: 'ready' | 'selecting' | 'checked' | 'complete';
  selectedOption: string | null;
  score: number;
  xp: number;
  hearts: number;
  streak: number;
  maxStreak: number;
}

type QuizAction =
  | { type: 'START'; questions: QuizQuestion[] }
  | { type: 'SELECT'; option: string }
  | { type: 'CHECK' }
  | { type: 'NEXT' }
  | { type: 'RESTART'; questions: QuizQuestion[] };
```

Implement `quizReducer` per the RESEARCH.md code example. Key rules:
- START: reset all state, 3 hearts, 0 xp, 0 streak, phase='selecting'
- SELECT: only works in 'selecting' phase, sets selectedOption
- CHECK: evaluates answer. Correct: score+1, xp+10, streak+1. Wrong: hearts-1 (min 0), streak reset to 0. Phase becomes 'checked'. Hearts NEVER block quiz (decorative only).
- NEXT: advance currentIndex. If last question, phase='complete'. Otherwise phase='selecting', clear selectedOption.
- RESTART: delegates to START.

**Keep existing fallback distractor logic:** The useEffect for fetching /api/distractors when vocabulary < 4 stays. The needsFallback, loadingFallbacks, fallbackMeanings useState hooks remain OUTSIDE the reducer (they're async fetch state, not quiz game state).

**Encouragement messages:**
```typescript
const CORRECT_MESSAGES = [
  "Sadhu! (Well done!)", "Ati uttamam! (Excellent!)",
  "Bahu sundaram!", "Samicinam! (Correct!)",
];
const WRONG_MESSAGES = [
  "Punah prayatnah (Try again)", "Not quite -- keep learning!",
];
const STREAK_MESSAGES: Record<number, string> = { 3: "On fire!", 5: "Unstoppable!" };

function getEncouragement(correct: boolean, streak: number): string {
  if (correct) {
    if (STREAK_MESSAGES[streak]) return STREAK_MESSAGES[streak];
    return CORRECT_MESSAGES[Math.floor(Math.random() * CORRECT_MESSAGES.length)];
  }
  return WRONG_MESSAGES[Math.floor(Math.random() * WRONG_MESSAGES.length)];
}
```
