**File 1: src/lib/analysis/ocr.ts**

Replace the entire `extractTextFromImage` function. The changes:

1. Change `createWorker("script/Devanagari")` to `createWorker("san")` -- Tesseract.js v7 uses ISO 639-3 language codes, not script paths. "san" is the code for Sanskrit which loads Devanagari-trained data. (Fixes OCR-01)

2. Add a 30-second AbortController-based timeout wrapping `worker.recognize()`. Implementation:
   - Create `const controller = new AbortController()` and `const timeoutId = setTimeout(() => controller.abort(), 30000)` before the recognize call
   - Since `worker.recognize()` does not natively accept an AbortSignal, wrap the call in `Promise.race()` with an abort-triggered rejection:
     ```
     const abortPromise = new Promise<never>((_, reject) => {
       controller.signal.addEventListener("abort", () => {
         reject(new Error("OCR timed out after 30 seconds. Try a clearer image or paste text directly."));
       });
     });
     const result = await Promise.race([worker.recognize(imageBuffer), abortPromise]);
     ```
   - Clear timeout with `clearTimeout(timeoutId)` after race resolves
   - In the finally block: call `controller.abort()` (to clean up listener if not already aborted), then `await worker.terminate()` (Fixes OCR-02)

3. Keep the existing `result.data.text.trim()` return value unchanged.

The full function signature remains: `export async function extractTextFromImage(imageBuffer: Buffer): Promise<string>`

**Change 2: Add useEffect for Object URL cleanup (OCR-03)**
Add a `useEffect` hook that depends on `preview`. When `preview` changes or the component unmounts, revoke the old Object URL:

```tsx
useEffect(() => {
  return () => {
    if (preview) {
      URL.revokeObjectURL(preview);
    }
  };
}, [preview]);
```

Place this hook right after the existing useState/useRef declarations, before the `handleFile` function.

Also, in `handleFile`, before calling `setPreview(URL.createObjectURL(file))`, revoke the existing preview if one exists:
```tsx
if (preview) {
  URL.revokeObjectURL(preview);
}
setPreview(URL.createObjectURL(file));
```

**Change 3: Add elapsed time counter during OCR extraction**
Add a new state variable after the existing state declarations:
`const [elapsedSeconds, setElapsedSeconds] = useState(0);`

Add a useEffect that runs an interval timer while `isUploading` is true:
```tsx
useEffect(() => {
  if (!isUploading) {
    setElapsedSeconds(0);
    return;
  }
  const intervalId = setInterval(() => {
    setElapsedSeconds((prev) => prev + 1);
  }, 1000);
  return () => clearInterval(intervalId);
}, [isUploading]);
```

Update the loading indicator JSX from:
```tsx
{isUploading && (
  <p className="mt-2 text-sm text-ink-600 animate-pulse">
    Extracting text...
  </p>
)}
```
to:
```tsx
{isUploading && (
  <p className="mt-2 text-sm text-ink-600 animate-pulse">
    Extracting text... ({elapsedSeconds}s)
  </p>
)}
```

**Note:** The `handleFile` function references `preview` state. To avoid stale closure issues, wrap `handleFile` in `useCallback` with `[preview]` dependency, OR access preview via a ref. The simplest approach: use a ref alongside the state:

`const previewRef = useRef<string | null>(null);`

In `handleFile`, use the ref for revocation:
```tsx
if (previewRef.current) {

previewRef.current = newUrl;

```

Update the cleanup useEffect to use the ref:
```tsx
useEffect(() => {
  return () => {
    if (previewRef.current) {
      URL.revokeObjectURL(previewRef.current);
      previewRef.current = null;
    }
  };
}, []);
```
This unmount-only cleanup with the ref avoids the stale closure issue entirely.
