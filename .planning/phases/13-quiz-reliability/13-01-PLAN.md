**1. vocabularyPopulator.ts -- Stem case normalization (per QUIZ-01):**

On line 48, change `stem: word.morphology.stem,` to `stem: word.morphology.stem.toLowerCase(),` so the stored stem is always lowercase.
```typescript
const distractorPool = allVocab
```

```typescript
const options = [correctAnswer, ...distractors];
// Safety: remove any distractor that somehow matches correctAnswer
const safeOptions = options.filter((opt, idx) => idx === 0 || opt !== correctAnswer);
// Re-pad if filtering removed options
while (safeOptions.length < 4) {
  safeOptions.push(item.stem !== correctAnswer ? item.stem : `(${item.iast})`);

shuffleArray(safeOptions);
```

Replace the existing `shuffleArray(options)` call on line 94 with the above block, and use `safeOptions` in the question push on line 96-104 instead of `options`.

```typescript
while (distractors.length < 3) {
  distractors.push(item.stem !== correctAnswer ? item.stem : `(${item.iast})`);
```

Fix the unsafe type cast in storableToCard() in src/lib/quiz/srs.ts (per STAB-02).

**Current bug (line 61):**
```typescript
last_review: data.last_review ? new Date(data.last_review) : undefined as unknown as Date,
```
The `undefined as unknown as Date` is a dangerous double-cast that hides a type mismatch. When `last_review` is undefined (new cards that have never been reviewed), this cast tells TypeScript it is a Date when it is actually undefined, which can cause runtime errors in ts-fsrs scheduling.

**Fix:**
Replace line 61 with a proper conditional that preserves the undefined value without casting:
```typescript
last_review: data.last_review ? new Date(data.last_review) : undefined,
```

Then update the return type annotation. The ts-fsrs Card type defines `last_review` as `Date`. Since we want to return undefined when there is no review, we need to cast the entire return object. Change the function signature to explicitly type the return:
```typescript
export function storableToCard(data: StorableCard): Card {
  return {
    due: new Date(data.due),
    stability: data.stability,
    difficulty: data.difficulty,
    elapsed_days: data.elapsed_days,
    scheduled_days: data.scheduled_days,
    reps: data.reps,
    lapses: data.lapses,
    learning_steps: data.learning_steps,
    state: data.state,
    last_review: data.last_review ? new Date(data.last_review) : undefined,
  } as Card;
}
```
The `as Card` cast on the return object is safe because ts-fsrs internally handles undefined last_review for new cards. This is a single, intentional cast at the boundary vs the previous double-cast on a single field.
