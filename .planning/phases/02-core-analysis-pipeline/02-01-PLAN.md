    - Each word in schema has: original (Devanagari), iast, sandhi_type enum, is_compound boolean, morphology object, contextual_meaning
    - Morphology object includes: stem, word_type enum, optional vibhakti/vacana/linga (nominals), optional dhatu/gana/lakara/purusha (verbs)
    - Samasa sub-schema includes: compound, is_compound, samasa_type enum (tatpurusha/dvandva/bahuvrihi/avyayibhava/karmadharaya/dvigu/none), components array
    - buildAnalysisPrompt(text) returns a string containing the input text and instructions for sandhi, samasa, morphology, and meaning analysis
    - TypeScript types (AnalyzedWord, SandhiInfo, MorphologyInfo, SamasaInfo, AnalysisResult) are defined and exported
  </behavior>
  <action>
    1. Run `npm install ai @ai-sdk/xai zod` to add AI SDK dependencies.

    2. Create `src/lib/analysis/types.ts` with TypeScript interfaces:
       - `SandhiInfo`: { sandhi_type: 'vowel'|'consonant'|'visarga'|'none'; sandhi_rule?: string }
       - `MorphologyInfo`: { stem: string; word_type: 'noun'|'verb'|'adjective'|'indeclinable'|'participle'|'pronoun'; vibhakti?: string; vacana?: 'ekavacana'|'dvivacana'|'bahuvacana'; linga?: 'pullinga'|'strilinga'|'napumsakalinga'; dhatu?: string; gana?: number (1-10); lakara?: string; purusha?: string }
       - `SamasaInfo`: { compound: string; is_compound: boolean; samasa_type?: 'tatpurusha'|'dvandva'|'bahuvrihi'|'avyayibhava'|'karmadharaya'|'dvigu'|'none'; components?: { word: string; iast: string; meaning: string; role: string }[] }
       - `AnalyzedWord`: { original: string; iast: string; sandhi: SandhiInfo; morphology: MorphologyInfo; samasa?: SamasaInfo; contextual_meaning: string; inria_validated: boolean; inria_grammar: string | null }
       - `AnalysisResult`: { input_text: string; words: AnalyzedWord[]; raw_llm_output: unknown }
       - `MeaningSource`: 'dictionary' | 'ai' | 'both'
       - `EnrichedWord`: extends AnalyzedWord with { mw_definitions: string[]; apte_definitions: string[]; meaning_source: MeaningSource }

    3. Create `src/lib/analysis/schemas.ts` with Zod schemas mirroring the types above. Follow the exact schema structure from the research doc (FullAnalysisSchema with nested MorphologySchema, SamasaSchema). These schemas are passed to `Output.object()` for structured LLM output.

    4. Create `src/lib/analysis/prompts.ts` with `buildAnalysisPrompt(text: string): string`. The prompt must:
       - Instruct the LLM as an expert Sanskrit grammarian (vaiyakarana)
       - Request sandhi splitting with type identification and rule citation
       - Request samasa decomposition with type classification (all 6 types)
       - Request morphological analysis (vibhakti/vacana/linga for nominals, dhatu/gana/lakara for verbs)
       - Request contextual meaning for each word in the passage
       - Require IAST transliteration for all Sanskrit terms
       - Use low temperature guidance (mention deterministic output desired)
       - Include examples of each sandhi type (vowel: a+i=e, consonant: t+j=jj, visarga: aH+vowel)
  </action>

    - When INRIA stem index has a match, inria_validated is true and grammar_info is populated
    - When INRIA has no match, inria_validated is false and inria_grammar is null
    - Pipeline returns AnalysisResult with words array containing INRIA-validated morphology
    - Tests mock generateText to return a fixed BG 1.1 analysis fixture for deterministic testing

  <action>
    1. Create `src/lib/analysis/morphology.ts`:
       - `enrichWithStemIndex(iast: string): { inria_validated: boolean; inria_grammar: string | null }` -- queries `lookupByStem(iast, 'iast')` and returns validation result
       - If stems found, return first match's grammar_info and validated=true
       - If no stems found, return null grammar and validated=false

    2. Create `src/lib/analysis/sandhi.ts`:
       - Re-export sandhi-related types from types.ts for module cohesion
       - `parseSandhiFromLLM(llmWord: z.infer<typeof WordSchema>): SandhiInfo` -- extracts sandhi info from LLM response into typed SandhiInfo

    3. Create `src/lib/analysis/samasa.ts`:
       - `parseSamasaFromLLM(llmWord: z.infer<typeof WordSchema>): SamasaInfo | undefined` -- extracts samasa info, returns undefined for non-compounds

    4. Create `src/lib/analysis/pipeline.ts`:
       - `analyzeText(text: string): Promise<AnalysisResult>` -- main orchestrator
       - Step 1: Call `generateText({ model: xai('grok-3-mini'), output: Output.object({ schema: FullAnalysisSchema }), prompt: buildAnalysisPrompt(text), providerOptions: { xai: { reasoningEffort: 'high' } } })`
       - Step 2: Map over output.words, for each word: parse sandhi, parse samasa, enrich with INRIA stem index
       - Step 3: Return typed AnalysisResult with all enriched words and raw LLM output
       - Handle errors: throw if output is null/undefined
